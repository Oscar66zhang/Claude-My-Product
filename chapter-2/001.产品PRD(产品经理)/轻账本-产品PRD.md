# 轻账本 — 个人财务管理 PRD

> 版本：v1.0.1 ｜ 日期：2026-05-22 ｜ 状态：初稿

---

## 一、产品概述

### 1.1 产品定位

**产品名称**：轻账本
**类型**：个人移动端记账工具
**一句话描述**：用最少的操作，记录每一天的每一笔消费，随时随地了解自己的钱花在了哪里。

### 1.2 背景与动机

现代人消费场景碎片化（外卖、订阅、扫码支付），传统记账方式（Excel、纸质）难以坚持。轻账本解决的核心问题是：**让记账变得足够简单，快到30秒内完成**，从而帮助用户真正养成记账习惯，而不是半途而废。

### 1.3 目标用户

- 有记账意愿但无法坚持的人群（25–40岁上班族）
- 希望了解自己消费结构、优化支出的用户
- 不需要复杂财务报表，只求清晰了解"钱去哪了"的用户

### 1.4 成功指标

| 指标 | 目标 |
|------|------|
| 单次记账操作完成时间 | ≤ 30 秒 |
| 次日留存率 | ≥ 40% |
| 30日留存率 | ≥ 15% |
| 用户平均每天主动打开次数 | ≥ 1.5 次 |

## 二、核心功能需求

### 2.1 记账功能（最高优先级）

**功能描述**：用户可在3步内完成一笔记账：选择分类 → 输入金额 → 保存

**操作流程**：
1. 首页点击「+」按钮，进入快速记账页面
2. 默认展示今日日期，支持修改
3. 选择支出分类（图标+文字，如「餐饮」「交通」「购物」等）
4. 输入金额（数字键盘，支持小数）
5. 可选填写备注
6. 点击保存，返回首页

**分类体系**（预设分类，支持后续扩展）：

| 大类 | 子分类示例 |
|------|-----------|
| 餐饮 | 早餐、午餐、晚餐、下午茶、零食 |
| 交通 | 打车、地铁、公交、停车、油费 |
| 购物 | 服装、电子产品、日用品、化妆品 |
| 居住 | 房租、水电煤、物业、维修 |
| 娱乐 | 电影、游戏、旅游、演出 |
| 医疗 | 门诊、药品、体检 |
| 其他 | 社交、捐赠、意外丢失 |

**收入记录**：同支出操作流程，额外标记「收入」类型，分类示例：工资、奖金、理财收益、兼职、红包。

### 2.2 流水记录列表

- 按日期倒序展示消费记录
- 支持按月份切换筛选
- 每条记录显示：时间、分类图标+名称、金额、备注（如有）
- 支持左滑删除某条记录
- 点击记录可编辑修改

### 2.3 月度账单总览

- 展示当月总支出、总收入、结余（总收入 - 总支出）
- 各分类支出金额及占比（柱状图/饼图）
- 环比上月：支出多了还是少了，幅度多少
- 支持切换查看历史月份

### 2.4 预算管理（次优先级）

- 用户可为每月设置总预算（如 5000 元）
- 首页显示「今日已花费」和「本月剩余可用」
- 当月支出超过预算时，以醒目的红色/橙色提示用户
- 超出预算后不影响继续记账，仅做预警提示

### 2.5 数据导出（后期考虑）

- 支持将账单数据导出为 CSV 文件
- 导出渠道：分享至微信/邮件/保存到本地

### 2.6 用户注册与登录（v1.0.1 新增）

**功能定位**：基础的身份认证功能，确保用户数据隔离与安全。

#### 2.6.1 注册

- 用户输入用户名（4-20位字母/数字）和密码（6-20位）
- 用户名不允许重复，提交时后台校验
- 密码在传输和存储时使用 SHA-256 单向哈希（不分词、不加盐，简化实现）
- 注册成功后自动登录，并跳转至首页

#### 2.6.2 登录

- 用户输入用户名 + 密码
- 登录成功后读取本地数据，显示用户信息
- 登录状态需持久化（LocalStorage 存储 session token）
- 密码错误时提示「用户名或密码错误」，不区分具体是哪个字段错误（防枚举攻击）

#### 2.6.3 登出

- 在「我的」页面提供「退出登录」按钮
- 清除本地 session token，显示登录页

#### 2.6.4 游客模式

- 未登录用户可正常使用记账功能（数据存储在本地 anonymous session）
- 游客登录后，可选择：合并游客数据 / 覆盖 / 取消（不允许）
- 避免因强制登录导致用户流失

#### 2.6.5 用户界面入口

- 首页顶部显示当前用户名（如未登录显示「未登录」）
- 点击用户名区域，可跳转至登录/注册页

## 三、用户界面设计要求

### 3.1 整体风格

- **设计语言**：简洁、轻量，不走重报表风格
- **配色方案**：主色调选用柔和的蓝绿色系（代表信任与平静），背景为浅灰白，支出显示为橙红色系，收入显示为蓝绿色系
- **字体**：系统默认字体，中文优先思源黑体 / 苹方
- **圆角与阴影**：适度圆角（8–12px），轻阴影突出卡片层次

### 3.2 页面结构

| 页面 | 核心元素 |
|------|---------|
| **登录页** | 用户名输入框、密码输入框、登录按钮、注册入口 |
| **注册页** | 用户名输入框、密码输入框、确认密码输入框、注册按钮、登录入口 |
| **首页/今日** | 日期标题、今日支出汇总、快捷记账按钮(+)、近期记录列表、用户名区域 |
| **记账页** | 日期选择、分类网格（图标+文字）、金额输入框、备注输入框、保存按钮 |
| **账单总览** | 月份切换、总收支结余卡片、分类占比图表、支出趋势 |
| **预算页** | 月预算设置、当月进度条、超支预警 |
| **我的/设置** | 数据导出、退出登录、关于页 |

### 3.3 导航设计

底部 Tab 导航，共 4 个入口：

```
[首页] [账单] [预算] [我的]
```

- **首页**：默认落地页，显示今日概况和快速记账入口
- **账单**：月度流水与报表
- **预算**：预算设置与进度
- **我的**：设置与导出

### 3.4 交互细节

- 记账完成后，以 Toast 提示「已记录」，并自动返回首页
- 底部数字键盘：输入金额时实时显示，无须点击确认输入框
- 空状态设计：首页无记录时显示引导插画 + 文案「记下第一笔吧」
- 加载状态：页面切换时使用骨架屏，不展示转圈加载

## 四、数据结构

### 4.1 核心数据模型

**Record（记账记录）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 记录唯一标识 |
| date | string (YYYY-MM-DD) | 记账日期 |
| type | enum | expense / income |
| category | string | 分类 ID |
| amount | number | 金额（正数，单位：元，精确到分） |
| remark | string (optional) | 备注，最长 100 字 |
| createdAt | timestamp | 创建时间 |
| updatedAt | timestamp | 更新时间 |

**Category（分类）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 分类唯一标识 |
| name | string | 分类名称 |
| icon | string | 图标名称（使用 Lucide 或 Emoji） |
| type | enum | expense / income |
| order | number | 展示排序 |

**Budget（预算）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 记录唯一标识 |
| month | string (YYYY-MM) | 预算所属月份 |
| amount | number | 月预算金额 |
| createdAt | timestamp | 创建时间 |

**User（用户）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string (UUID) | 用户唯一标识 |
| username | string | 用户名（4-20位字母/数字，唯一） |
| passwordHash | string | 密码 SHA-256 哈希值 |
| createdAt | timestamp | 注册时间 |

### 4.2 本地存储策略

- 所有数据存储于设备本地（使用 LocalStorage）
- 用户信息与记账数据分开存储
  - `lightAccountBook_user`: 当前登录用户信息（不含密码）
  - `lightAccountBook_records`: 当前用户的所有记账记录
  - `lightAccountBook_budget`: 当前用户的预算数据
- 不强制要求用户注册账号（支持游客模式）
- 提供数据备份导出功能（CSV）作为数据安全兜底
- 数据隔离：同一设备不同用户的数据严格隔离

## 五、非功能性需求

### 5.1 性能要求

- 冷启动时间（首次打开到可交互）：≤ 2 秒
- 记账操作响应：≤ 100ms
- 列表滑动帧率：≥ 50fps
- 离线可用：所有功能在无网络环境下均可正常使用

### 5.2 兼容性

- 首发平台：iOS 13+、Android 8+
- 推荐使用 React Native 或 Flutter 开发
- 适配主流全面屏机型（刘海的安全区域处理）

### 5.3 隐私与安全

- 不申请不必要的权限（不需要通讯录、位置等）
- 数据不上传至服务器（除非用户主动开启云同步）
- 数据导出时不做任何追踪

## 七、上线后的迭代方向（记录供参考）

以下为 v1.0 之后的潜在功能方向，不影响本期scope：

- **多设备同步**：账号体系 + 云端存储
- **多设备数据同步**：登录后自动同步本地数据至云端
- **周期性账单**：自动记录每月固定支出（房租、订阅）
- **数据洞察**：基于历史数据给出"本月消费建议"
- **多人模式**：家庭账本，支持多人共同记录
- **语音记账**：说"今天午饭花了30块"自动识别写入
